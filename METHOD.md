# Method

## 为什么不是全量 AI 逐条标注

大规模评论数据常见问题：

- 行数多，逐条 AI 标注成本和耗时高。
- 评论里有反讽、调侃、转折、正负混合和低信息文本。
- 纯关键词规则容易误判。
- 直接全量 AI 虽然语义强，但结果不一定稳定，且难复核。

因此本 toolkit 推荐混合流程：

```text
脱敏清洗
-> 抽样 AI 标注
-> 建立标签口径
-> 分类器迁移到全量
-> 低置信/边界/反讽/随机样本复核
-> 修正口径或分类器
-> 输出统计和质量摘要
```

## 有标签体系

如果用户已经给出标签，比如：

```text
正向期待 / 中性讨论 / 负向抵触 / 风险担忧 / 无关低信息
```

agent 应先把标签写成判断标准、包含/排除规则、边界案例，再抽样标注和迁移。

## 没有标签体系

如果是新议题，不应直接套正/中/负。先抽样观察真实表达，必要时使用 k-means/关键词发现表达簇。

k-means 的作用是发现“相似文本堆”，不是最终情感分类器。最终标签需要根据业务问题和样本含义命名、合并和确认。

## 质量控制

至少披露：

- 原始行数；
- 脱敏/清洗后行数；
- 去重后行数；
- 有效样本分母；
- AI 标注样本量；
- 分类器迁移样本量；
- 低置信、边界、反讽、随机复核样本量；
- 标签口径是用户提供还是从样本探索建立。
